# GymMate Backend

Бэкенд приложения GymMate на [PocketBase](https://pocketbase.io) (v0.39.3, SQLite).
Запускается в Docker, на проде за reverse-proxy [Caddy](https://caddyserver.com)
с автоматическим HTTPS (Let's Encrypt).

## Что внутри

- **PocketBase** — БД (SQLite), REST/Realtime API и админ-панель. Схема описана
  миграциями в `pb_migrations/` и применяется автоматически при старте.
- **Caddy** — reverse-proxy с автоматическими TLS-сертификатами (используется только на проде).
- Описание модели данных — в [`SCHEMA.md`](SCHEMA.md) (+ справочный PostgreSQL DDL в [`schema.sql`](schema.sql)).

## Требования

- Docker и Docker Compose
- `make` (опционально — все команды можно вызывать и напрямую через `docker compose`)

## Настройка

1. Скопируйте файл с переменными окружения и заполните его:

   ```sh
   cp .env.example .env
   ```

   | Переменная     | Назначение                                                                 | Прод            | Локально |
   | -------------- | -------------------------------------------------------------------------- | --------------- | -------- |
   | `DOMAIN_URL`   | Домен для Caddy (например `api.gymmate.app`), на него выпускается сертификат | обязательно     | не нужно |
   | `DOMAIN_EMAIL` | Email для Let's Encrypt                                                     | обязательно     | не нужно |
   | `PB_HOST`      | Адрес, на котором публикуется PocketBase. По умолчанию `127.0.0.1` (только localhost). Выставьте `0.0.0.0`, чтобы открыть доступ из локальной сети | **не задавать** | опционально |

   > На проде `PB_HOST` оставляйте дефолтным — PocketBase не должен торчать наружу
   > напрямую, доступ идёт только через Caddy.

## Запуск

### Локальная разработка

Поднимает только PocketBase (без Caddy), с логами в консоль:

```sh
make dev
# = docker compose --progress=plain up pocketbase --build
```

После старта доступно:

- Админ-панель: <http://127.0.0.1:8090/_/>
- API: <http://127.0.0.1:8090/api/>

При первом запуске PocketBase попросит создать аккаунт суперпользователя
(через админ-панель или командой `pocketbase superuser create`).

Чтобы открыть доступ с других устройств в сети (например с телефона), выставьте
в `.env` значение `PB_HOST=0.0.0.0` и обращайтесь по IP машины: `http://<ваш-ip>:8090`.

### Прод

Поднимает PocketBase + Caddy в фоне (detached). Caddy сам выпустит TLS-сертификат
для `DOMAIN_URL`:

```sh
make run
# = docker compose up --build -d
```

После этого API доступно по `https://<DOMAIN_URL>/api/`, админка — `https://<DOMAIN_URL>/_/`.

### Полезные команды

```sh
docker compose logs -f pocketbase   # смотреть логи
docker compose down                 # остановить и удалить контейнеры
docker compose up --build -d        # пересобрать и перезапустить
```

## Миграции

Схема коллекций задаётся JS-миграциями в `pb_migrations/` и применяется
автоматически при старте контейнера (`pocketbase serve` подхватывает
`--migrationsDir=./pb_migrations`).

Если применяете миграции на уже запущенном инстансе — после этого
**перезапустите сервер**: PocketBase кэширует схему коллекций в памяти.

Подробности о структуре данных — в [`SCHEMA.md`](SCHEMA.md).

## Данные

- Данные PocketBase хранятся в `./pb_data` (примонтировано в контейнер, в git не попадает).
- Каталоги `pb_data/`, `backups/` и `.env` исключены из репозитория — храните их вне git.
